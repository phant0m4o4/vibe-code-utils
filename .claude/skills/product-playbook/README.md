# product-playbook

产品全流程陪跑 skill:从"一个想法"到"规模化稳定运行",陪你走完 18 个阶段,一次推进一步,每步落地产物、过关卡再继续。

## 这是什么

一个 Claude Code [skill](https://docs.claude.com/en/docs/claude-code/skills)。它把产品经理的全流程方法论(概念 → 机会判断 → 用户研究 → 需求 → MVP → PRD → 技术评审 → 研发 → 测试 → 灰度 → 数据验证 → 运营 → 迭代 → 复盘)变成 Claude 可执行的陪跑流程,核心理念八个字:**想清楚、做出来、跑数据、再迭代。**

它**不是**自动写完所有文档的生成器。它的价值在于:定位你当前在哪个阶段、只推进这一个阶段、把结论落成文件、用关卡(gate)拦住没想清楚的地方,然后停下等你确认。

## 什么时候会自动触发

你不需要手动调用它。只要在对话里聊产品层面的事,Claude 就会自动启用——哪怕你没提"流程"二字。典型场景:

- 抛一个产品/App/工具的想法("我想做个 X""这个点子靠谱吗")
- 评估值不值得做、做用户调研、定需求、写 PRD、设计 MVP
- 准备上线 / 灰度 / 发布,分析上线后的数据效果
- 排迭代优先级、做项目复盘

如果你只是想让它别介入(比如只想随手讨论),直说即可。

## 它怎么干活

固定四步,每步的理由都写在 SKILL.md 里:

1. **定位阶段**——判断你在 18 阶段的哪一步,从那里切入,不从头重跑。
2. **读当前阶段清单**——只读 `references/checklists.md` 里对应的那一节。
3. **产物落地、过 gate**——结论写进目标项目的 `docs/product/` 下对应文件,逐条核对关卡。
4. **停下等确认**——一次只推进一个阶段,不替你一口气生成全套文档。

开始前它会先看"按规模裁剪"表:自用小工具不会被逼着走千万级流程。

## 怎么启用

- **只在本仓库生效**:保持现在的位置 `.claude/skills/product-playbook/` 即可。
- **在所有项目生效**(更符合它的用途):把整个目录拷到全局 skills 目录。

  ```bash
  cp -r .claude/skills/product-playbook ~/.claude/skills/
  ```

## 目录结构

```
product-playbook/
├── SKILL.md            # 运行主干:四步工作流 + 18 阶段总览 + 各阶段 gate + 按规模裁剪
├── README.md           # 本文件(给人看的导览)
├── references/
│   └── checklists.md   # 各阶段详细提问清单,运行时按阶段按需加载
└── source-article.md   # 原文存档,仅供人工参考,skill 运行时不引用
```

`SKILL.md` 是给 Claude 读的运行说明书,`README.md` 是给人读的导览——两者职责不同,不要把运行规则往这里抄。
